[01] USER_GUIDE.NFO

This macro extracts RAW REJECTION DATA from BTS without triggering IT alarms. Follow these steps to generate your dataset.

STEP_01: Open your IBM PComm Session. Navigate to the main rejection list (e.g., TFST or TREX).

STEP_02: Open the Excel Workbook. Locate the Control Panel at the top (Rows 1-4).

STEP_03: Verify Speed Settings (ms):

  • B1 (Tab): 200
  • B2 (Drill): 850
  • B3 (Back): 850
  • B4 (Page): 1200

STEP_04: Execute RunMainframeScrape. A small bridge will initiate communication. DO NOT TOUCH THE KEYBOARD while the bridge is active.

STEP_05: Once "Scrape Complete" appears, your raw data will be populated starting at Row 6. Column A is the row, Column B contains the Special Instructions.

[02] CODE_ANATOMY.EXE

Understanding the PCOMM_BRIDGE architecture. This is how we bypass standard limitations.

THE ANCHOR (RegEx)

We don't hardcode rows. We use a pattern-seeking missile:

regEx.Pattern = "_ [A-Z]{3,4}"
// Finds the first underscore followed by 3-4 letters.
// This tells the script exactly where the data starts on ANY screen.

THE DRILL-DOWN (Logic)

The script "Deep-Dives" into each entry to find the rejection reason:

objPS.SendKeys "[pf2]" ' Enter the detail screen
i = InStr(1, buf, "SPEC INS:", 1) ' Locate the label
fRow = ... + 1 + 1 ' Shift 1 row DOWN to grab the text

THE CIRCUIT BREAKER (Paging)

How do we know when to stop? We compare the "Before" and "After" snapshots:

If buf = lastScreen Then Exit Do
// If the screen didn't change after pressing PA1, we've hit the end.
